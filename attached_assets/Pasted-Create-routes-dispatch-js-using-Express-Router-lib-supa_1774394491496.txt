Create routes/dispatch.js using Express Router, 
lib/supabase.js, and lib/twilio.js.

This file handles all SMS command processing for 
CommandField. Build it in two parts:

PART 1 — THE COMMAND PARSER (lib/parser.js)

Create lib/parser.js with a function called 
parseCommand(text) that uses deterministic 
rule-based matching with regex extraction.

It should detect and return a command object for 
each of these patterns:

OWNER/MANAGER COMMANDS:
- assign: /^assign\s+(.+?)\s+to\s+@(\S+)\s+on\s+(\S+)\s+at\s+(\S+(?:\s*[ap]m)?)\s*-\s*(.+?)\s*-\s*client\s+@(.+)/i
  returns { type: 'assign', job_type, worker_tag, 
  date, time, address, customer_tag }

- reassign: /^reassign\s+(.+?)\s+to\s+@(\S+)/i
  returns { type: 'reassign', job_ref, worker_tag }

- cancel: /^cancel\s+job\s+(.+)/i
  returns { type: 'cancel', job_ref }

- jobs_today: /^jobs\s+today$/i
  returns { type: 'jobs_today' }

- jobs_worker: /^jobs\s+@(\S+)/i
  returns { type: 'jobs_worker', worker_tag }

- kpi_today: /^kpi\s+today$/i
  returns { type: 'kpi_today' }

- kpi_week: /^kpi\s+week$/i
  returns { type: 'kpi_week' }

- kpi_month: /^kpi\s+month$/i
  returns { type: 'kpi_month' }

- workers: /^workers$/i
  returns { type: 'workers' }

- customer_lookup: /^customer\s+@(.+)/i
  returns { type: 'customer_lookup', customer_tag }

- invoice_lookup: /^invoice\s+@(.+)/i
  returns { type: 'invoice_lookup', customer_tag }

- add_worker: /^add\s+worker\s+(.+?)\s+(\+?[\d\s\-\(\)]{10,})/i
  returns { type: 'add_worker', name, phone }

- add_customer: /^add\s+customer\s+(.+?)\s+(\+?[\d\s\-\(\)]{10,})\s+(.+)/i
  returns { type: 'add_customer', name, phone, address }

- note: /^note\s+job\s+(\S+)\s+(.+)/i
  returns { type: 'note', job_ref, note_text }

WORKER COMMANDS:
- start: /^start$/i or /^start\s+(.+)/i
  returns { type: 'start', job_ref: null or ref }

- done: /^done$/i or /^done\s+(.+)/i
  returns { type: 'done', job_ref: null or ref }

- on_my_way: /^on\s+my\s+way$/i
  returns { type: 'on_my_way' }

- issue: /^issue\s+(.+)/i
  returns { type: 'issue', description }

- my_jobs: /^my\s+jobs$/i
  returns { type: 'my_jobs' }

- clock_in: /^clock\s+in$/i
  returns { type: 'clock_in' }

- clock_out: /^clock\s+out$/i
  returns { type: 'clock_out' }

- help: /^help$/i
  returns { type: 'help' }

If no pattern matches:
  returns { type: 'unknown' }

Export parseCommand as default.

PART 2 — DISPATCH ROUTES

In routes/dispatch.js create:

POST /api/dispatch/sms-inbound
This is the Twilio webhook endpoint.
- Accepts Twilio webhook body: From, Body
- Looks up the sender phone number in public.profiles
  to identify who is texting
- If not found: reply "Your number is not registered 
  with CommandField. Contact your administrator."
- Gets their role and business_id from profiles
- Calls parseCommand(Body) from lib/parser.js
- Based on command type and role, executes the 
  appropriate handler (see handlers below)
- Returns Twilio TwiML response format:
  res.set('Content-Type', 'text/xml')
  res.send(`<?xml version="1.0"?>
  <Response><Message>${replyText}</Message></Response>`)

POST /api/dispatch/command
This is the dashboard text input endpoint.
- Accepts: { business_id, user_id, text, role }
- Calls parseCommand(text)
- Executes appropriate handler
- Returns: { success: true, command, result, reply }

GET /api/dispatch/log/:business_id
- Returns all dispatch_commands for that business_id
- Ordered by created_at descending
- Limit 50

COMMAND HANDLERS:
Build these as async functions inside dispatch.js:

handleAssign(parsed, business_id):
- Look up worker by matching name containing 
  worker_tag in public.workers
- Look up customer by matching name containing 
  customer_tag in public.customers  
- Parse date and time into a timestamp
- Create job in public.jobs
- Send SMS to worker:
  "New job: [job_type] at [address] on [date] 
  at [time]. Client: [customer_name]. 
  Reply START when on your way."
- Send SMS to customer:
  "[Business name] has scheduled [job_type] 
  for [date] at [time]. 
  Tech: [worker_name]. Questions? Reply here."
- Log to public.dispatch_commands
- Return summary text for reply

handleJobsToday(business_id):
- Query jobs scheduled for today
- Format as SMS-friendly list:
  "Jobs Today:\n1. [type] @[worker] - 
  [time] - [status]\n2. ..."
- If none: "No jobs scheduled for today."

handleKPI(business_id, period):
- Query KPI data for period (today/week/month)
- Format as SMS:
  "KPI [period]:\nScheduled: X\nIn Progress: X\n
  Completed: X\nRevenue: $X\nWorkers Active: X"

handleStart(sender_phone, business_id):
- Find most recent in_progress or scheduled job 
  for this worker
- Update status to in_progress, set started_at
- Notify owner via SMS:
  "[Worker] started [job_type] at [address]. [time]"
- Notify customer:
  "Your tech [name] is on the way!"
- Reply to worker: "Job started. Good luck!"

handleDone(sender_phone, business_id):
- Find most recent in_progress job for this worker
- Update status to completed, set completed_at
- Notify owner:
  "[Worker] completed [job_type] at [address]. 
  Reply INVOICE @[customer] to send invoice."
- Reply to worker: "Job marked complete. Great work!"

handleMyJobs(sender_phone, business_id):
- Find all scheduled/in_progress jobs for 
  this worker today
- Format as SMS list
- If none: "No jobs scheduled for you today."

handleHelp(role):
- Return formatted command list appropriate 
  for their role (owner commands vs worker commands)

handleUnknown():
- Return: "Command not recognized. 
  Reply HELP for available commands."
